Hey Hacker, and welcome!
iExec is launching the Vibe Coding Challenge, an online builder challenge to create use cases built on Nox, the confidential smart contract layer of iExec. On top of Nox, iExec just launched the Confidential Token: a wrapper that transforms any ERC-20 into a confidential, auditable asset.

This challenge is supported by ChainGPT, enabling builders to integrate AI-powered features such as smart contract generation, auditing and on-chain data insights into their applications.

Free API credits are available for participants.

What is Nox - Protocol Layer
Nox is a confidential computing layer that enables computations on encrypted data while preserving full DeFi composability. It combines on-chain smart contracts with off-chain Trusted Execution Environments (TEE) to process encrypted data without ever exposing plaintext on-chain.

With Nox, you build confidential smart contracts and tokens that stay fully composable with existing DeFi protocols. Your contracts can process encrypted inputs, execute computations privately, and maintain hidden balances, without asking users to change wallets or developers to rewrite contracts.

What is the Confidential Token - Product Layer

The Confidential Token is the first product built on Nox.

It is a reversible ERC-20 wrapper that enables hidden balances and transaction amounts while staying fully composable with existing DeFi protocols. Any ERC-20 wraps into its confidential equivalent.

Main Goal
Inspired by the vibe coding movement, this challenge encourages builders to experiment with AI-assisted development and rapid prototyping, and put the Nox protocol to work in real use cases.

The goal: turn ideas into working prototypes fast, and demonstrate what confidential and programmable financial logic unlocks for DeFi & RWA using vibe coding tools.

The Challenge
Build an application using NOX protocol & Confidential Tokens of the powered by iExec.

Your project should leverage Confidential Tokens to explore privacy-enabled use cases in areas such as DeFi, real-world assets (RWA) or tokenization.

The goal is to demonstrate how confidential and programmable financial logic can unlock new applications while preserving privacy and composability.

Participants are encouraged to experiment and build creative prototypes.

Builder Ideas
Need inspiration? Here are concrete project ideas you can build:

Confidential RWA & Tokenization Protocols:
Private Onchain stocks
Build tokenized equities (e.g. cAAPL, cTSLA) with confidential balances and transactions, leveraging compliant tokenization standards such as ERC-3643 and confidential token extensions like ERC-7984.
Private Securities & Commodities
Enable confidential tokenized equity with private ownership and allocation, while maintaining regulatory compliance using standards like ERC-3643.
Private Yield via T-Bills & Bonds
Enable institutional-grade DeFi & RWAs with confidential exposure to yield-bearing assets such as bonds and T-Bills, built on compliant tokenization frameworks like ERC-3643.
Confidential DeFi:
Confidential Vault
Build yield-generating vaults that protect strategy positions and capital allocations from copy-trading and MEV, using confidential tokens ERC-7984 and vault standards such as ERC-7540 to preserve privacy.
Private Lending/borrowing/yield
Build lending, borrowing, and yield primitives where positions and strategies remain private, leveraging confidential token standards such as ERC-7984.
Note: For ERC standards (ERC-3643, ERC-7540, ERC-7984) implementations must fully adhere to the specifications including correct integration of encrypted types when applicable. Partial implementations will not be considered valid.
